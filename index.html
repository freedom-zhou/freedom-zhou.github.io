Linux中Ctrl+Z和Ctrl+C的区别

Share

这两个命令都是中断命令，但是他们的作用却不一样。

Ctrl C是强制终止程序的执行并结束进程。

而Ctrl Z则只是中断任务的执行，但该任务并没有结束，它只是在进程中维持挂起的状态，用户可以使用fg/bg操作来继续前台/后台执行该任务。

fg命令会重启前台被中断的任务，bg命令则把被中断的任务放到后台执行。

例如：

当你用vim编辑一个文件时，如果此时需要使用shell执行别的操作，但是你又不想关闭vim，你可以简单的按下Ctrl Z，shell会将vim进程挂起，当你结束了那个shell操作之后，使用fg命令继续vim你的文件。

不同版本Linux系统的镜像源配置——解决软件包安装慢的问题

Ubuntu20 下配置镜像源

Ubuntu 安装软件的工具为apt
镜像源列表配置文件路径：/etc/apt/sources.list

亲测好用的镜像源
中国科学技术大学：
http://mirrors.ustc.edu.cn/ubuntu

若为带有图像界面的Ubuntu，可根据以下步骤自动选择最佳镜像源：
Software & Updates -> Ubuntu Software -> Download from: (点击下拉列表) -> Other ->Select Best Server
等待一段时间后，点击Choose Server，Ubuntu就自动配置好了当前访问速度最快的镜像源

Deepin 下配置镜像源

Deepin 安装软件的工具为apt
镜像源列表配置文件路径：/etc/apt/sources.list

Deepin15系统设置支持换源

Deepin20系统设置不支持换源了，因为换源可能会造成各种问题，所以官方去掉了这个功能，实在要换只能自己修改sources.list`文件

CentOS 下配置镜像源

CenOS 安装软件的工具为yum
镜像源列表配置文件路径：/etc/yum.repos.d/CentOS-Base.repo
